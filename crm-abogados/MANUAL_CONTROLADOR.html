Este manual es para ti, el operador de LexCRM — no para el abogado cliente. Explica el proceso real, paso a paso, para dar de alta a un nuevo cliente según el plan que contrató.

Restricción técnica real: hoy el sistema NO restringe automáticamente el Asistente IA por plan. LexCRM es una única aplicación multi-tenant: cualquier email en ALLOWED_EMAILS obtiene acceso a todas las funciones, incluida la IA, sin importar el plan pagado. La diferenciación es hoy comercial, no técnica. Mientras no se construya el gating (columna plan + chequeo en /api/ai/* y en la UI), debes avisar tú mismo al cliente Básico que la IA no está incluida en su plan, aunque el botón esté visible. No prometas a nadie que el límite ya está aplicado en el código — no lo está.

02

Proceso de alta de un nuevo cliente

Paso 1 — Llega el contacto

El prospecto escribe por WhatsApp desde la landing (wa.me/56979710838) con un mensaje prellenado que ya indica el plan elegido: "Hola, quiero contratar el plan Básico/Pro de LexCRM". Si viene del botón de prueba gratis, el mensaje dice "quiero probar LexCRM gratis 7 días" — acceso completo por una semana, con recordatorio manual tuyo para dar de baja si no contrata al terminar.

Paso 2 — Verificas el pago

Envías los datos de tu cuenta bancaria y confirmas la transferencia antes de activar el acceso (salvo trial gratuito).

Paso 3 — Autorizas su email

  1. Vercel → proyecto del CRM → Settings → Environment Variables.
  2. Edita ALLOWED_EMAILS y agrega el correo del cliente, separado por coma de los existentes.
  3. Guarda y ejecuta Redeploy (Deployments → los tres puntos del último deploy → Redeploy). Sin este paso el cambio no toma efecto.

Paso 4 — Envías las credenciales de entrada

  • Enlace: https://app.lexcrm.site
  • Instrucción: clic en "Registrarse", usando exactamente el correo autorizado en el paso 3.
  • El Manual de Usuario en PDF (entregable estándar, incluido en todos los planes).

Paso 5 — El cliente completa su perfil

Al primer ingreso, el sistema lo redirige obligatoriamente a Mi Perfil hasta que complete sus datos de contacto y cuenta bancaria (para emisión de honorarios). No puede usar el resto del CRM hasta hacerlo — es el comportamiento esperado, no un error.

Paso 6 — Queda operativo

Desde ahí el cliente ya puede cargar sus propios clientes y causas.

04

Registro de planes por cliente

El sistema no guarda qué plan contrató cada abogado — ALLOWED_EMAILS es solo una lista de correos, sin metadata de plan. Mientras no exista una columna plan en la base de datos, esa información vive únicamente en tu propio registro.

Qué llevar

Mantén una planilla simple (Google Sheets sirve) con estas columnas:

  • Email
  • Nombre del estudio / abogado
  • Plan (Básico / Pro)
  • Fecha de alta
  • Próximo cobro
  • Estado (activo / moroso / dado de baja)

Es tu única fuente de verdad para saber a quién cobrarle este mes y a quién limitar el uso de IA "de palabra" mientras el plan Básico no tiene bloqueo técnico.

05

Cambios de plan y bajas

Upgrade Básico → Pro

No requiere cambio técnico — el cliente ya tenía acceso a la IA de facto. Solo actualiza tu planilla y el nuevo monto de cobro.

Downgrade o cancelación

  1. Vercel → ALLOWED_EMAILS → elimina el correo del cliente.
  2. Redeploy.
  3. El cliente verá el banner "cuenta pendiente de activación" la próxima vez que intente entrar — no se le borran sus datos, solo pierde acceso. Sus datos quedan en la base de datos hasta que solicite eliminación formal (derecho de supresión, Ley 21.719).

Renovación mensual

No hay cobro automático — no hay pasarela de pago habilitada para Chile todavía. Debes verificar tú mismo, mes a mes, la transferencia de cada cliente activo según tu planilla. El sistema no genera ninguna alerta de pago vencido.
